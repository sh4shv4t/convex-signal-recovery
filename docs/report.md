# Technical Report: Entropy-Regularized Signal Recovery

## Motivation
Signal recovery often seeks parsimonious representations that explain observed data while avoiding overfitting. Minimizing $\sum_i x_i$ provides a convex surrogate that encourages low-complexity solutions and behaves similarly to $\ell_1$-style regularization when combined with feasibility constraints. By varying the constraint regime, we can study how geometric restrictions influence the feasible set, the quality of the recovered signal, and solver behavior in finite-dimensional settings.

## Problem Formulation
Let $A \in \mathbb{R}^{m \times n}$, $x \in \mathbb{R}^n$, and $b \in \mathbb{R}^m$. We study the convex program
$$
\min_{x \in \mathbb{R}^n} \sum_{i=1}^n x_i
$$
under three regimes:
1. **Equality constraints:** $A x = b$.
2. **Inequality constraints:** $A x \ge b$.
3. **Inequality with bounds:** $A x \ge b$ and $-1 \le x_i \le 1$ for all $i$.

The data are generated by sampling $A$ and $b$ from standard normal distributions and creating a sparse $x_{\text{true}}$ via a Bernoulli mask multiplied by Gaussian entries. The observation is $y = A x_{\text{true}} + b$.

## Constraint Regimes and Geometry
The equality case defines an affine subspace. When this subspace intersects the objective’s level sets, the optimizer is uniquely determined up to feasibility, but the solution may be unbounded if the objective descends along a feasible direction. The inequality case replaces the affine subspace with a polyhedral cone; this enlarges the feasible set and can admit many solutions with comparable objective values, potentially exposing unboundedness if the cone admits descent directions.

The bounded inequality case introduces a box constraint that intersects the polyhedral cone, producing a compact feasible region. Geometrically, this trims long rays that could lead to unbounded objectives, stabilizing the solution and enabling direct comparison of recovered signals across trials. The bounding constraints also restrict magnitude, which can reduce variance in the recovered vector while preserving sparsity trends.

## CVX Implementation Notes
Each case is implemented in CVX using `cvx_begin quiet`, a single decision variable $x$, and linear constraints. The objective remains identical across cases, enabling a clean comparison. The main driver reports solver status and objective values, while post-processing creates visual summaries and saves figures to the results folder.

## Expected Solver Behavior
In the equality case, feasibility is sensitive to the random draw of $A$ and $b$; CVX may report infeasibility if no exact solution exists. The inequality case can be feasible more often but may be unbounded if the feasible set contains directions that decrease the sum objective without bound. The bounded case is expected to be feasible and bounded more consistently, with a stable objective value and norm. Warning messages in the driver highlight any infeasible or unbounded outcomes so the user can interpret results accordingly.
